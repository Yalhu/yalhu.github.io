Mac 键盘快捷键

2020.6.11 星期四 18：26

基础

快捷键

快捷键可以在 设置-》键盘中找到，或者设置
Mac 键盘快捷键

command + H： 隐藏
command + control + Q: 锁屏
command + delete: 删除光标前到行首内容

同一应用不同窗口切换： Command+`/～
也可以四指上滑调出所有页面选择想要切换的页面。

command + 空格: 聚焦
option + command + 空格： 访达搜索窗口

手势/手指

熄屏

想快速熄屏，而不用关机或待机 control+shift+关机键（关机键需要等1.5s后按下）
或者设置触发角（左下角）屏幕进入睡眠

快速熄屏：control+shift+Eject。
睡眠状态的快捷键是：Option + Command + Eject。
锁屏： control+command + q

键盘差异

输入法：大小写/中英文 同一个键。 长按3s切换为大写字母，此时灯亮。

行首/行尾：command/fn + 方向箭 （上下是页头页尾。左右是行，如有空格不会跳到空格前面，而是改行文字前面）

delete(向后删除文字)：fn + 删除键

Finder/文件操作

删除： command + 删除键
移动： 直接拖动。（高版本可以设置3值拖动快捷键）
复制：command + c。 也可以在拖动的过程中 按住 option（此时会出现一个绿色的 + 号）

剪切：不能通过 command + x 剪切文件/文件夹（文本内容可以）
Command + Option + V 作用相当于Windows里的文件剪切。
在其它位置上对文件复制（Command-C），在目的位置按下这个快捷键，文件将被剪切到此位置

打开文件/夹： Command + O 打开所选项。在Mac里打开文件不像Windows里直接按Enter
；第二种方法是 command + ↓。
另外 command + ↑可以进入当前目录的上一级目录。

Space 快速查看选中的文件，也就是预览功能
切换显示隐藏文件： shift + command + .

新建文件

mac 不和windows 一样可以有右键新建文件的快捷方式，或是其它右键快捷方式 New File.app。
可以通过 Automator（自动操作）建立 服务 或者 应用程序。其中应用程序可以添加到finder快捷栏，通过鼠标点击操作。

或者一些拓展工具：New File Menu、New File Menu Free。
或者直接准备和一个空文件，直接复制/粘贴

修改应用程序图标

还可以利用一些软件来删除（有些软件卸载可能不干净）

DiskCatalogMaker for Mac,Disk Graph for mac,Disk Xray
CleanMyMac X for mac(Mac垃圾清理系统优化工具)

$_PS: 上面的软件目前可能还用不到，安装软件又要占据空间。删除效果页没有尝试过，目前就是因为磁盘空间小（128g），而不是其它原因。直接删除文件就好

浏览器

刷新：command + r
历史记录：command + y

滚动一屏: fn + 箭头
页首/页尾：command + 箭头

编辑器

vscode

删除： command + shift + k （原生的设置，windows中只是手动设置了command + d）
替换：command + option + f
command + shift + f： 打开侧边栏的搜索
跳行： control + g

插件

markdown 的删除线 快捷键无效： option + s

Mac电脑没声音了怎么办？解决Mac上声音问题
选择正确的音频设备; 查看音量等 基本操作。

重置核心音频

在Mac上，coreaudiod是启动核心音频的启动守护程序。

如果声音停止工作或变得失真（破裂或嘈杂），则重新启动coreaudiod流程应该可以解决问题。

1） 一种是打开“ 活动监视器”，在右上角的搜索框中键入coreaudiod，然后单击“ 强制退出”按钮以手动终止该过程。
2） 另一种方法是启动终端并键入以下命令：sudo killall coreaudiod 按回车键，输入管理员密码，然后再次检查您的声音。

在极少数情况下，如果它没有重新启动，您可能根本听不到任何声音。在这种情况下，请关闭并重新启动Mac。
如果目前无法重新启动，请改用以下Terminal命令：
sudo launchctl start com.apple.audio.coreaudiod

Mac 外接显示器时无法用键盘调节音量？这个方法能够帮助到你 | 一日一技
键盘上的音量控制键不能调节（显示器扬声器的）音量了，菜单栏的音量滑块也不行。这是因为 HDMI、DisplayPort 和 Thunderbolt 等接口传输的都是带有固定音量的数字音频信号，而能调节音量的音频信号属于模拟信号，因此只有外接显示器将数字信号转换为模拟信号后才能通过键盘调节音量。

从原理上说，Soundflower 的主要作用是将音频信号重新定向。例如，你想在 iTunes 中播放音乐的同时，想用 QuickTime 录制这一段音频，就可以使用 SoundFlower 作为 iTunes 音频的输出，同时将 SoundFlower 作为 QuickTime 的音频输入。而在重定向过程中，SoundFlower 就可以把数字音频信号转换为模拟信号，再调节增益后转换为数字信号，变相实现音量调节，当然这些过程也会伴随有音质的损失。

Ds_Store 文件

DS_Store 是用来存储这个文件夹的显示属性的：比如文件图标的摆放位置。删除以后的副作用就是这些信息的失去。（当然，这点副作用其实不是太大）

尽管这些文件本来是为Finder所使用，但它们被设想作为一种更通用的有关显示设置的元数据存储，诸如图标位置和视图设置。 例如，在Mac OS X 10.4 “Tiger”中.DS_Store包含了一文件夹的所有文件的Spotlight注释。然而，在Mac OS X 10.5 “Leopard”中这种方式又被更改了，注释（现称为关键字）被改成存储在了扩展文件属性中。

和别人交换文件（或你做的网页需要上传的时候）应该把 .DS_Store 文件删除比较妥当，因为里面包含了一些你不一定希望别人看见的信息（尤其是网站，通过 .DS_Store 可以知道这个目录里面所有文件的清单，很多时候这是一个不希望出现的问题）　　

操作

1、禁止.DS_store生成：打开 “终端” ，复制黏贴下面的命令，回车执行，重启Mac即可生成。
defaults write com.apple.desktopservices DSDontWriteNetworkStores -bool TRUE

2、恢复.DS_store生成：
defaults delete com.apple.desktopservices DSDontWriteNetworkStores

3、删除 所有目录的“.DS_store”文件：在terminal中输入:
sudo find / -name ".DS_Store" -depth -exec rm {} \;
